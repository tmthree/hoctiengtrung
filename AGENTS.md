# AGENTS.md — Framework-Specific Warnings

## Next.js App Router (v15)

### Critical Rules
- **Server Components are default** — add `"use client"` only when needed (hooks, event handlers, browser APIs)
- **Do NOT mix** Server and Client component patterns in same file
- **Server Actions** must have `"use server"` directive at top of file or function
- **Metadata** exports only work in Server Components (layout.tsx, page.tsx)
- `cookies()`, `headers()` are async in Next.js 15 — must `await` them

### Common Mistakes
- Importing a Client Component hook (useState, useEffect) in a Server Component → build error
- Using `redirect()` in a try/catch block → redirect throws, will be caught
- Passing non-serializable props (functions, dates) from Server to Client Component
- Using `fetch()` in Server Actions when you can use Prisma directly

### Route Groups
- `(marketing)`, `(auth)`, `(app)`, `(admin)` are route groups — parentheses mean NO URL segment
- Each has its own layout.tsx for different shells

### Prisma + Neon
- Use `@prisma/adapter-neon` for serverless connection pooling
- Always use singleton pattern for Prisma client (`lib/db.ts`)
- Neon cold starts: first query may be slow — use connection pooling URL

### Better Auth
- Auth handler at `app/api/auth/[...all]/route.ts`
- Server-side: `auth.api.getSession({ headers: await headers() })`
- Client-side: use `authClient` from `lib/auth-client.ts`

### next-intl
- All pages under `[locale]/` dynamic segment
- Middleware handles locale detection and redirects
- Use `useTranslations('namespace')` in Client Components
- Use `getTranslations('namespace')` in Server Components

### shadcn/ui
- Components in `src/components/ui/` are generated by CLI
- Do NOT edit them directly — customize via className prop or wrapper components
- Add new components: `npx shadcn@latest add <component-name>`
